# Hermes Memory Protocol

At startup always read:

1. spec.md
2. hot.md
3. AI/contracts
4. AI/logs

Before any task:
- Understand current architecture
- Check current state
- Review active goals

After every task:
- Update spec.md
- Update hot.md
- Save important logs
- Document architecture decisions
- Save errors and fixes

Never:
- Forget architecture decisions
- Modify contracts without logging
- Claim task completed without updating memory
